# WebStyle：蓝色科技军工指挥舱规范

## 适用范围

本文档约束 `web/public` 控制台前端：

- `web/public/dashboard.html`
- `web/public/dashboard.css`
- `web/public/dashboard.js`

不改变后端 API、WebSocket 协议、场景 JSON 格式或日志 schema。

## 风格定位

整体方向是“蓝色科技感军工指挥舱”：深海军蓝/石墨黑底，电蓝、冰蓝、青蓝为主高亮，少量琥珀用于警告，红色用于异常。页面应该像实时战术态势与任务控制台，而不是普通后台、纸面仪表盘或单纯换色的暗色页面。

避免回到紫粉霓虹赛博、橄榄绿单色军工、米色纸张或传统浅色后台风格。

## 布局规范

主布局保持稳定工作台结构：

- 左侧为主监视区，占约 3 份宽度
- 右侧为控制和日志区，占约 1 份宽度
- 左侧上下分区：场景面板在上，Agent Canvas 战术态势图在下
- 小屏可以上下堆叠，但不能遮挡日志、控制区、场景面板或 Canvas 交互

场景面板保持内嵌，不做悬浮窗。场景折叠时，上方场景面板收缩，下方 Agent Canvas 自动扩展。

## 视觉系统

统一使用 CSS 变量管理颜色、阴影和边框。主色应围绕蓝色科技体系：

| 变量 | 用途 |
| --- | --- |
| `--bg` / `--bg-deep` | 页面和 Canvas 深色底 |
| `--panel` / `--panel-strong` | 指挥舱面板 |
| `--blue` / `--cyan` / `--ice` | 主高亮、HUD、链路 |
| `--green` | 正常、运行、友方状态 |
| `--amber` / `--orange` | 警告、任务焦点、广播 |
| `--red` | 错误、失败、异常 |
| `--muted` | 次级标签、时间戳 |

面板应有硬边框、切角、扫描线、低亮度网格或刻度质感。不要把大区块做成嵌套卡片；卡片只用于必要的局部信息块、下拉菜单或悬浮详情。

## Canvas 规范

Canvas 是主视觉，应呈现“战术态势图”：

- 背景包含蓝色雷达网格、方位刻度、扫描线和低亮度噪点
- Agent 不是普通圆点，应有目标锁定环、角标、状态识别色和活动扫描弧
- 关系链路使用细线、虚线、脉冲和适度光效表达强弱与方向
- 报文轨迹像战术信号脉冲，广播可用琥珀色，普通消息用蓝/青色
- 空状态应保持战术监视语气，不破坏 Canvas 构图

Canvas 交互必须保持：缩放、平移、拖 Agent、双击 reset、悬停 Agent 信息面板。

## 悬浮窗规范

Agent 悬停详情面板使用毛玻璃效果：

- 半透明蓝黑玻璃背景
- `backdrop-filter` / `-webkit-backdrop-filter` 模糊和饱和增强
- 细亮边、内侧高光、切角和字段分区
- 内容优先可读，不能为了透明感降低文字对比

毛玻璃只用于 Agent 悬停详情面板。日志下拉菜单保持工具菜单质感，主面板保持装甲/仪表面板质感。

## 日志规范

右侧日志是“战报终端”：

- 时间、来源、类型、消息主体层级清晰
- 来源标签像频道/频段标识，但不能影响排查效率
- 长文本点击展开，默认单行省略，不撑破布局
- 新日志可以闪入，但不能频繁触发布局抖动
- 高频日志数量应裁剪，避免无限堆积动画事件

## 修改约束

HTML 可以重构表现结构，但必须保留功能入口和关键 ID：

- `scene-selector`
- `agent-canvas`
- `tooltip`
- `scene-panel`
- `fp-iframe`
- `fp-title`
- `log-viewer`
- `log-fields`
- `log-entries`

CSS 响应式规则集中维护，`prefers-reduced-motion` 必须收口。JS 的 Canvas 绘制逻辑保持模块化，避免把后端协议、日志 schema 或仿真流程揉进视觉代码。

## 验收清单

修改 `web/public` 前端后，应至少确认：

- 页面主色调是蓝色科技军工指挥舱
- 左侧场景面板在上，Agent Canvas 在下
- 场景折叠后 Canvas 扩展正常
- 右侧控制区和战报日志区不重叠、不遮挡
- Agent tooltip 有毛玻璃效果且可读
- Canvas 缩放、平移、拖 Agent、双击 reset 正常
- 日志长文本不溢出、不撑破布局
- 小屏布局不遮挡核心操作
