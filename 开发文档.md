# AI Agent 仿真运行平台

> **版本**: 0.1.0 | **技术栈**: Python / FastAPI / React / Docker

AI Agent 仿真运行平台是一个**企业级多 Agent 仿真与推演平台**。用户通过自然语言或 JSON 定义场景角色和目标，平台自动创建 Agent 实例，驱动 LLM 决策，实现多角色博弈推演，并在可视化地图上实时呈现。

---

## 一、核心能力

| 能力 | 描述 |
|------|------|
| **场景定义** | 自然语言剧本 / JSON 结构化场景 (角色、目标、关系、事件) |
| **Agent 管理** | 自动创建、生命周期管理、按角色/技能发现 |
| **LLM 决策** | 每个 Agent 独立 LLM 大脑，根据角色和目标自主决策 |
| **Agent 通信** | 消息总线路由，Agent 间可通过 LLM 决策互发消息 |
| **容器化运行** | 每个 Agent 在独立子进程中运行，通过消息总线通信 |
| **可视化** | Campus Atlas 地图 + Agent 实时位置 + 关系拓扑图 + 悬停面板 |

## 四、运行模式

每个 Agent 在独立子进程中运行，通过 HTTP 消息总线通信。支持 Docker 和 subprocess 两种启动方式（Docker 可用时自动切换，不可用时回退子进程）。进程隔离确保单个 Agent 崩溃不影响其他。

---

## 五、Agent 注册流程

每个 Agent 启动后向消息总线注册，建立路由表：

```text
场景定义 (script_json)
  │
  └─ server.py: 创建 Agent × N
       │
       ├─ 内存模式: agent.set_comm(LocalBus) → 直存对象引用
       │
       └─ 容器模式: subprocess("agent_server.py")
             ├─ 环境变量注入身份 (AGENT_ID, AGENT_NAME, AGENT_CORE_GOAL...)
             ├─ 启动 uvicorn 监听独立端口 (:8101, :8102...)
             └─ 向消息总线注册: POST :9000/register
                   参数: agent_id=role_a, name=大型电信公司CEO, url=:8101

消息总线注册表:
  {
    "role_a":        "http://localhost:8101",   ← 按 agent_id
    "大型电信公司CEO": "http://localhost:8101",   ← 按 name 别名
    "role_b":        "http://localhost:8102",
    ...
  }
```

## 六、Agent 通信流程

Agent 间通过统一通信接口 (`CommLayer`) 完成，内存模式和容器模式共用同一接口。

### 通信接口

```text
CommLayer
  ├─ .send(from_id, from_name, target, content) → bool
  ├─ .broadcast(from_id, from_name, content) → bool
  └─ .register_agent(agent_id, name, url) → None

LocalBus  (内存) — agent.inbox.append() 直接投递
RemoteBus (容器) — POST :9000/relay → HTTP 转发
```

### 容器模式通信链路

```text
Round N:
═══════
  server.py: runtime.run_round()
    │
    ├─ POST :8101/decide  → Agent A 决策
    │     └─ Brain.decide(inbox, context)
    │           ├─ 系统提示 (角色 + 目标 + 秘密 + 可用行动)
    │           ├─ 已知 Agent 列表 + 收件箱消息
    │           ├─ LLM API 调用 → 决策
    │           └─ Action { type:"send_message", target:"role_b", content:"谈判条件" }
    │
    ├─ POST :8101/act  → Agent A 执行
    │     └─ comm.send("role_a", "CEO", "role_b", "谈判条件")
    │           └─ RemoteBus → POST :9000/relay
    │
    └─ (并行) 对所有 Agent 同时 decide + act

消息总线转发:
═══════════
  POST :9000/relay
    ├─ 查注册表: agent_registry["role_b"] → http://localhost:8102
    └─ POST :8102/message → Agent B inbox
          {
            from: "大型电信公司CEO",
            content: "谈判条件"
          }

Round N+1:
══════════
  POST :8102/decide → Brain 读取 inbox → LLM 分析 → 决定回复
    └─ Action { type:"send_message", target:"role_a", content:"接受条件" }
```

## 七、场景定义

### JSON 结构化场景 (script_json)

```json
{
  "script_json": {
    "scenario_metadata": {
      "title": "电信招标战",
      "background_rules": "多家企业角逐国家级电信合同..."
    },
    "role_slots": [
      {
        "slot_id": "ROLE_A",
        "identity": "大型电信公司CEO",
        "core_goal": "在招标中获得合同，确保市场份额达到30%",
        "initial_assets": { "funding": 200, "influence": 90 },
        "hidden_secret": "与评标人有私人关系，可在关键时刻施加影响",
        "action_space": ["出价", "游说评标人", "威胁竞争对手", "隐瞒技术不足"]
      }
    ],
    "initial_relationship_matrix": {
      "links": [
        { "from": "ROLE_A", "to": "ROLE_B", "relation_type": "利益竞争", "value": -70 },
        { "from": "ROLE_A", "to": "ROLE_C", "relation_type": "潜在合作", "value": 30 }
      ]
    },
    "event_triggers": [
      { "turn": 5, "event_name": "突发合规检查", "impact": "所有企业须提交额外报告" }
    ]
  }
}
```

`role_slots` 直接映射为 Agent：`identity` → 名称，`core_goal` → 任务，`action_space` → 技能，`hidden_secret` → 秘密。关系矩阵渲染为拓扑连线。

### 自然语言剧本

```text
"3个侦察兵搜索地图，2个分析员评估威胁，1个指挥官制定方案"
```

LLM 自动解析为 Agent 定义和工作流步骤。

## 八、LLM 集成

每个 Agent 拥有独立的 LLM 大脑，决策循环：

```
观察 (inbox) → 推理 (LLM) → 决策 (Action) → 执行 (act)
```

### 支持的后端

| 后端 | 配置 |
|------|------|
| Anthropic Claude | `ANTHROPIC_API_KEY` |
| OpenAI / 兼容 | `OPENAI_API_KEY` |
| DeepSeek | `LLM_API_KEY` + `LLM_MODEL=deepseek-chat` |

### 系统提示词

Agent 使用场景 JSON 中的 `background_rules` 作为系统提示词，结合角色的 `identity`、`core_goal`、`action_space` 等信息构建决策上下文。没有独立的角色提示词，所有行为准则来自场景定义。

## 九、前端可视化

### Dashboard (web/dashboard.*)

主控制台，功能：Campus Atlas 地图嵌入、场景选择与运行、仿真日志面板、Agent 悬停详情面板。

### Campus Atlas (web/tactical-map/)

React 应用，渲染华为智慧园区地图 + Agent 实时位置 + 关系拓扑连线。

**Agent 和地图在同一 Canvas 上渲染**，共用坐标系，缩放/拖拽天然一致。

**关系拓扑图**：绿色实线 = 合作、红色虚线 = 竞争，中点标注关系类型，线条透明度反映关系强度。

**悬停面板**：显示 Agent 身份、状态、核心目标、隐藏秘密、资产、可选行动。
