# ADR-020：所有受管文件统一经过 FileManager

## 状态

已接受，分阶段迁移中。

## 决定

剧本、日志、PCAP、capture manifest、experiment manifest、quality、analysis 和归档文件统一由 `FileManager` 管理物理文件生命周期。

业务模块保留各自的业务语义，但不得继续新增直接文件写入、读取、压缩、解压、下载、可见性或删除实现。

## 统一职责

`FileManager` 统一提供：

- 安全路径解析；
- 原子写入与追加写入；
- 文本、二进制和流式读取；
- ZIP 压缩和受限安全解压；
- 下载资源准备；
- 可见性状态；
- 大小和 SHA-256；
- 物理删除和资源生命周期元数据。

## 领域边界

- 剧本管理负责剧本结构和引用校验；
- `LogManager` 负责日志 schema、规范化和内存索引；
- 抓包模块负责 `tcpdump` 生命周期；
- `pcap_reader` 负责协议解析；
- `experiment_manifest` 负责实验来源和质量语义；
- `FileManager` 不理解上述业务内容，只管理文件资源。

## 禁止回退

- 不得在业务 API 中新增直接 `open`、`Path.read_text`、`Path.write_text`、`unlink`、`rmtree` 或 `ZipFile` 操作受管业务文件；
- 不得让不同业务模块分别维护可见性元数据；
- 不得把隐藏实现为重命名、移动或删除文件；
- 不得直接向外部 API 返回物理路径；
- 不得绕过统一路径策略解压用户归档；
- 不得在哈希不一致时继续提供下载；
- 不得把文件基础设施状态复制到多个独立注册表形成多重权威。

## 写入和安全要求

- 普通写入和注册表写入采用临时文件加 `os.replace`；
- 相对路径不得包含绝对路径或 `..`；
- 解压拒绝路径逃逸和符号链接；
- 解压必须限制条目数、单文件大小和总大小；
- 失败的临时文件和临时目录必须清理；
- 目录下载必须先生成受管归档资源。

## 可见性语义

- 可见性只影响默认列表和普通下载；
- 隐藏资源仍然存在，允许具有显式内部权限的业务流程访问；
- 显示操作只恢复 `visible=true`；
- `deleted` 资源不能通过显示操作恢复。

## 迁移规则

1. 先新增统一基础设施和测试；
2. 优先迁移日志文件，删除 `.log_visibility.json` 独立机制；
3. 再迁移剧本上传、解压、持久化和下载；
4. 最后迁移 PCAP、manifest、分析结果和 bundle；
5. 每个领域迁移完成后，删除原直接文件操作代码，不保留双轨兼容；
6. 新功能从本 ADR 生效起必须直接使用 `FileManager`。

## 观测方式

统一文件注册表记录 `resource_id`、所有者、资源类型、存储根、相对路径、逻辑名称、媒体类型、可见性、状态、大小、SHA-256 和时间。业务日志可以引用 `resource_id`，但不得记录对外可见的物理路径。
