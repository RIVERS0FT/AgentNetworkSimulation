# ADR-020：所有受管文件统一经过 FileManager

## 状态

已接受并完成基础设施迁移。

## 决定

剧本、Skill 源文件、日志、PCAP、capture manifest、experiment manifest、quality、analysis、packet sample 和归档文件统一由 `FileManager` 管理物理文件生命周期。

业务模块保留各自的业务语义，但不得新增独立的写入、读取、压缩、解压、下载、可见性、哈希或删除实现。

## 权威入口

- 核心：`agent_network/file_management/file_manager.py`
- 资源模型：`agent_network/file_management/models.py`
- 项目根配置：`agent_network/file_management/defaults.py`
- 日志接入：`agent_network/file_management/log_integration.py`
- 剧本接入：`agent_network/scene_storage.py`

统一注册表为 `file-registry.v1`，默认位于共享 PCAP 卷的 `.file_registry.json`。控制面和 Agent 容器使用同一文件，注册表操作必须使用跨进程文件锁和原子替换。

## 领域边界

- 剧本管理负责剧本结构和引用校验；
- Skill source 负责 allowlist 和 Skill 包语义；
- `LogManager` 负责日志 schema、规范化和内存索引；
- 抓包模块负责 `tcpdump` 生命周期；
- PCAP 子系统负责协议解析；
- `experiment_manifest` 负责实验来源和质量语义；
- `FileManager` 只管理文件资源和安全存储。

`tcpdump` 是允许的外部文件生产者，但目标必须来自受管根，生成后必须注册或刷新资源。

## 禁止回退

- 不得在业务 API 中直接使用 `open`、`Path.read_text`、`Path.write_text`、`unlink`、`rmtree` 或 `ZipFile` 操作受管业务文件；
- 不得重新建立 `.log_visibility.json` 或其他领域私有可见性注册表；
- 不得把隐藏实现为重命名、移动或删除；
- 不得向外部 API 返回物理路径；
- 不得绕过统一路径策略解压用户归档；
- 不得在哈希不一致时继续提供下载；
- 不得让多个进程以无锁方式覆盖共享注册表；
- 不得让已隐藏或已删除的资源在目录扫描时被静默重新显示；
- 不得为旧直接文件路径保留双轨兼容入口。

## 安全与失败语义

- 路径必须相对受管根，拒绝绝对路径、`..`、逃逸和符号链接；
- 写入使用临时文件与 `os.replace`；
- 解压限制数量和大小，失败时清理全部临时数据；
- 目录必须归档后下载；
- `ready`、`failed`、`deleted` 与 `visible` 分别表达生命周期和展示状态；
- 批量业务操作必须逐项返回结果，不因单项失败丢失其他结果。

## 观测方式

统一注册表记录 `resource_id`、所有者、资源类型、受管根、相对定位、逻辑名称、媒体类型、可见性、状态、大小、SHA-256 和时间。业务日志和 manifest 可以引用 `resource_id`，不得记录对外可见的物理路径。
