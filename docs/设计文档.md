# AI Agent 仿真运行平台设计文档

> 技术栈：Python / FastAPI / Docker Compose / 原生 Web Dashboard

本项目用于模拟多 Agent 运行时状态、通信行为和网络流量。平台通过场景文件夹定义角色、容器实例、Skill allowlist、Tool allowlist 和通信拓扑；`srv` 负责仿真编排，`bus` 负责通信转发与通信权限强制校验，容器内 Agent 负责 Skill.md SOP 加载、Tool 注册和后端原生推理执行。

详细边界见：`docs/AgentRuntimeBoundary.md`。

---

## 一、当前职责边界

| 组件 | 职责 | 禁止事项 |
|---|---|---|
| `srv` | 场景解析、Agent 编排、容器分配、通信矩阵下发、日志聚合 | 不 import 场景 `tools.py`；不读取 Skill.md 正文；不执行 ToolRegistry |
| `bus` | Agent 注册表、消息转发、通信矩阵保存、`/relay` 权限校验 | 不用 prompt 替代权限控制；不做模糊目标匹配 |
| Agent 容器 | 接收 `/run` 上下文，启动后端适配器，加载 Skill.md SOP，启动 MCP server | 不依赖 `srv` 注册 Skill/Tool |
| BackendAdapter | OpenCLAW / Claude Code 运行入口，注入 SOP context，接入 MCP | 不把 Skill 名称当作可执行工具 |
| MCP server | 读取场景 `tools.py`，注册 ToolRegistry 中允许的原子 Tool | 不把 Skill.md 包装成工具 |

## 三、运行链路

```text
1. POST /api/simulations/setup
   ├─ srv 读取 meta_and_roles.json
   ├─ srv 读取 instances_and_skills.json 中的 skill_refs / tool_refs
   ├─ srv 读取 network_topology.json 构建关系图
   └─ srv 创建 AgentDef / AgentRegistry 展示对象

2. POST /api/simulations/launch
   ├─ srv 为每个 Agent 分配容器
   ├─ srv 将 agent_id -> container_url 注册到 bus
   ├─ srv 将 network_topology 生成的 comm_matrix 下发到 bus
   ├─ srv 每轮调用容器 /run
   └─ srv 收集 BackendAdapter 返回结果和日志

3. Agent 容器 /run
   ├─ agent_server.py 接收 scene_key / skill_refs  / allowed_tools
   ├─ BackendAdapter 在容器内读取 /app/scenes/<scene>/skills/*.md
   ├─ Skill.md 作为 SOP/context 注入后端任务 payload
   ├─ BackendAdapter 启动容器本地 MCP server
   ├─ MCP server 读取 /app/scenes/<scene>/tools.py
   ├─ ToolRegistry.list_tools() 中允许的工具注册为 MCP tools
   └─ OpenCLAW / Claude Code 通过 MCP 调用工具

4. Agent 通信
   ├─ Agent 调用 send_message / broadcast MCP tool
   ├─ comm.py POST 到 bus /relay
   ├─ bus 检查 from_id -> to 是否在 comm_matrix 中
   ├─ 允许则转发到目标容器 /message
   └─ 不允许则返回 403 并记录 policy denied 事件
```

## 四、Skill.md 与 Tool 规范

### 4.1 Skill.md 是 SOP

Skill.md 只描述“什么时候使用、如何判断、建议调用哪些原子工具”。它不会被注册成可执行函数。

```markdown
---
name: query_dashboard
description: 获取面板全量数据
version: 1.0
tools:
  - query_dashboard_tool
---

# Skill: query_dashboard

## 何时使用
当需要查看当前场景面板状态时使用。

## 执行步骤
1. 调用 query_dashboard_tool。
2. 检查返回数据。
3. 结合当前任务生成结论。
```

### 4.2 tools.py 是原子工具入口

场景工具必须通过 `ToolRegistry` 暴露：

```python
class ToolRegistry:
    @staticmethod
    def list_tools():
        return ["query_dashboard_tool"]

    @staticmethod
    def execute(name: str, **kwargs):
        if name == "query_dashboard_tool":
            return {"ok": True}
        raise ValueError(f"unknown tool: {name}")
```

MCP server 只注册 `allowed_tools` 中允许的工具，外加系统原子工具 `send_message` / `broadcast`。

## 五、通信权限控制

通信权限由 `network_topology.json` 生成，不依赖 prompt。

```json
{
  "sub_networks": [{
    "edges": [
      {"source": "CEO", "target": "CTO", "direct_chat": true},
      {"source": "CEO", "target": "DEV_LEAD", "direct_chat": false}
    ]
  }]
}
```

规则：

- `direct_chat=false` 的边不会进入 bus 通信矩阵；
- 单播目标不在矩阵中时，bus `/relay` 返回 403；
- 广播只投递给矩阵允许目标；
- bus 真实路由键只使用 `agent_id`；展示名只作为精确 alias。

## 六、镜像与启动

`docker-compose.yml` 包含基础服务与 Agent runtime 镜像服务：

| 服务 | 镜像 | 说明 |
|---|---|---|
| `srv` | server 镜像 | 控制面 |
| `bus` | message-bus 镜像 | 通信面 |
| `ag-o1` | `agentnetwork-ag-o1` | OpenCLAW Agent runtime 镜像 / 池容器 |
| `ag-c1` | `agentnetwork-ag-c1` | Claude Code Agent runtime 镜像 / 池容器 |

启动：

```bash
docker compose config
docker compose up -d --build
```

验证：

```bash
curl http://localhost:9000/policy
curl http://localhost:9000/agents
```

运行场景后，应看到 `policy_loaded=true`，且无权限通信会被 bus 拒绝。

## 七、关键设计约束

- `srv` 只传 allowlist，不读取 SOP 正文。
- Skill.md 不等于 Tool，不注册为函数。
- Tool 必须是 `tools.py` 中的原子动作。
- MCP server 是容器内 Tool 注册边界。
- 通信权限必须在 bus 强制执行。
- 日志、网络抓包、应用层事件只做观测，不改变权限边界。
