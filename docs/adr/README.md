# AgentNetwork 架构决策记录索引

本目录只保存架构决策记录（ADR）。需求、详细设计、开发指南和实施记录分别位于 `../requirements/`、`../design/`、`../guides/` 和 `../records/`。

## 编号规则

- ADR 使用永久、唯一、三位数字编号 `ADR-NNN`；
- 当前决策统一标为 `ADR-001`～`ADR-037`，一个编号只对应一个文件；
- 已废弃或被替代的 ADR 保留原编号和正文，不删除、不复用；
- 新决策从 `ADR-038` 开始递增；
- 修订或替代关系必须同时写入新旧 ADR，并更新本索引。

## 通用变更规则

1. 每次工作先阅读 `../README.md`、ADR-030、相关 ADR、详细设计和代码。
2. 全部修改都必须留下设计记录；代码、配置、脚本、测试、部署与设计文档在同一 PR 中保持一致。
3. 当前处于初期开发阶段：优先直接迁移，不为旧实现增加无需求兼容分支。
4. 删除架构后，同时删除文档、Compose、测试和 Dashboard 中的旧引用。
5. 新增字段、接口或后台任务时，写明所有权、生命周期、失败语义和观测方式。
6. 完成后检查完整 diff，确保没有任务范围外的 `scenes/` 文档修改。
7. 记录验证命令、预期结果和实际结果；无法执行时如实记录风险和后续责任人。
8. 设计记录、实现映射和验证证据齐全后才允许合并。

Agent 通信继续遵守 [ADR-024](ADR-024-CommManager统一A2A通信与禁止广播.md) 和 [ADR-025](ADR-025-Agent任务下发与A2A回调.md)：不得恢复 `DirectBus`、中心 relay 或广播路径，多目标必须顺序发送。
