# ADR-037：存量剧本一次性迁移到 v2 目录合同

- 状态：已接受并实施
- 日期：2026-07-23
- 关联：[ADR-036](ADR-036-剧本目录采用Env主数据合同.md)

## 背景

ADR-036 已定义新的剧本目录合同，但仓库仍保留一批使用 `meta_and_roles.json`、`instances_and_skills.json`、`network_topology.json` 和根目录 `tools.py` 的旧剧本。继续同时维护两套文件会导致任务、环境数据和 Tool 注册位置持续分叉。

## 决策

仓库内所有同时具备三份旧核心 JSON 的剧本目录一次性迁移为：

```text
Agents.json
topology.json
env.py
skills/
tools/
panel.html  # 可选
```

迁移规则固定如下：

1. `Agents.json` 保存 Agent 身份、能力引用和 Agent 执行任务。
2. 原 `instances_and_skills.json.tasks` 原样映射到对应 Agent；缺少显式任务时，以该 Agent 的 `core_goal` 生成一个 `<agent>-primary` 主任务。
3. `env.py` 保存标题、描述、全局规则、初始状态、共享数据、旧元数据扩展、角色扩展和可选业务拓扑。
4. 每个剧本生成一个 `scene-complete` 剧本级任务，依赖该剧本全部 Agent 任务。
5. `network_topology.json.topology` 迁移到 `topology.json.topology`，链路语义不改变。
6. 根目录 `tools.py` 移动到 `tools/legacy.py`；如果旧 Tool 实现在 `skills.py` 且完整覆盖 `tool_refs`，则复制到 `tools/legacy_skills.py`。
7. 缺失的 Skill Markdown 入口生成显式标注为“迁移生成”的占位文档，保留引用但不伪造业务实现。
8. `business_topology.json` 和 `scale_config.json` 的数据写入 `ENV["environment"]`，原辅助文件暂不删除。
9. 新文件完成结构校验后删除三份旧核心 JSON 和根目录 `tools.py`。
10. 迁移脚本必须幂等；旧文件删除后再次运行不得改写已迁移剧本。

## 数据所有权

- Agent 执行任务归 `Agents.json` 所有。
- 剧本级完成条件归 `env.py.scene_tasks` 所有。
- 旧角色和实例中无法直接映射到 v2 Agent 字段的数据归 `ENV["environment"]` 所有。
- Tool 可执行代码归 `tools/*.py` 所有。

## 失败语义

遇到以下情况时整个迁移命令失败，不提交半完成结果：

- 旧核心文件不是合法 JSON 对象；
- Agent 集合为空；
- Agent 引用的 Tool 在 `tools.py` 或 `skills.py` 中没有对应注册；
- 生成后的任务依赖、Skill、Tool 或拓扑引用不完整；
- `env.py` 无法通过字面量解析。

## 实现映射

- `scripts/migrate_scenes_v2.py`：扫描、转换、验证和生成实施记录。
- `tests/test_scene_v2_migration.py`：验证任务迁移、主任务合成、环境数据保留、Tool 移动、Skill 补全和幂等性。
- `docs/records/实施记录-旧剧本迁移到v2.md`：由迁移脚本根据实际结果生成。

## 验证

自动化执行顺序：

```bash
PYTHONPATH=. python scripts/migrate_scenes_v2.py \
  --root scenes \
  --delete-legacy \
  --report docs/records/实施记录-旧剧本迁移到v2.md
PYTHONPATH=. python -m pytest \
  tests/test_scene_v2_migration.py \
  tests/test_scene_env_contract.py
PYTHONPATH=. python -m compileall scripts agent_network scenes
```

严格校验调用必须向 `SceneValidatorV2` 传入已解析的绝对剧本根路径，避免 Skill 与 Tool 入口在不同工作目录下产生错误的相对路径判断。

迁移后的每个剧本必须通过 `SceneValidatorV2`，且目录中不再存在三份旧核心 JSON 或根目录 `tools.py`。
