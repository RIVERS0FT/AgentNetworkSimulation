# 剧本严格校验与读取设计

> 状态：**已实施**。剧本运行时只接受 `agentnetwork-scene.v2`，旧三文件格式只允许由一次性迁移脚本作为离线输入处理。

## 1. 权威目录合同

单个可运行剧本必须包含：

```text
scenes/<scene_key>/
├── Agents.json
├── topology.json
├── env.py
├── skills/
│   ├── <skill_ref>.md
│   └── <skill_ref>/SKILL.md
├── tools/
│   └── *.py
└── panel.html            # 可选
```

`Agents.json`、`topology.json`、`env.py` 缺少任意一个时，该目录不是可发现、可读取或可上传的剧本。

## 2. 单一读取入口

所有运行时读取必须经过 `SceneStorage`：

- `SceneStorage.list_scenes()`：只列出具备三份 v2 入口的目录，标题来自 `ENV["metadata"]["title"]`；
- `SceneStorage.details()`：返回 Agent、Skill、Tool、任务、拓扑、环境和校验结果；
- `SceneStorage.build_definition()`：构造仿真运行使用的 `SceneDefinition`；
- `SceneManager`：只编排列表、详情、上传、批量解析、下载和删除，不重复解析文件；
- 正式 HTTP 接口通过 `SceneManager` 读取；执行模块公开的列表和详情辅助函数由 `agent_network.api` 包入口统一覆盖，并委托 `SceneStorage`。

文件注册表中即使残留旧目录记录，只要目录不具备完整 v2 入口，也不得出现在列表或读取结果中。

## 3. 文件所有权

### 3.1 `Agents.json`

根字段固定为 `agents`。每个 Agent 持有：

- `name`、`role`、`background`、`core_goal`；
- `backend`；
- `skill_refs`、`tool_refs`、`native_capabilities`；
- Agent 范围的 `tasks`。

Agent 任务的 `agent_id` 由所在 Agent 推导，校验后以 `scope="agent"` 写入统一任务集合，并把任务目标投影到 `AgentDef.tasks`。

### 3.2 `env.py`

`env.py` 必须只有可选模块文档字符串和一次顶层 `ENV` 赋值。平台使用 AST 与 `ast.literal_eval` 静态读取，禁止 import、函数调用、函数或类定义以及其他可执行语句。

`ENV` 根字段固定为：

- `metadata`：`title`、`description`；
- `environment`：剧本规则、初始状态、共享数据和领域扩展；
- `scene_tasks`：剧本范围任务。

剧本任务以 `scope="scene"` 写入统一任务集合，不绑定 Agent、Skill 或 Tool。

### 3.3 `topology.json`

根字段固定为 `topology`。每条链路只允许：

- `endpoint_a`、`endpoint_b`、`channel_id`；
- `delay_ms`、`jitter_ms`、`loss_pct`、`rate_mbit`。

链路端点必须引用已定义 Agent，`channel_id` 在剧本内唯一。

### 3.4 Skill 与 Tool

- Skill 入口支持 `skills/<ref>.md` 和 `skills/<ref>/SKILL.md`；
- Tool 从 `tools/**/*.py` 递归发现；
- Tool ID 在剧本内唯一；
- 注册调用引用的函数必须在同一源文件定义；
- Agent 和 Agent 任务引用必须落在其已授权 Skill、Tool 集合内。

## 4. 统一任务图

Agent 任务和剧本任务共享任务 ID 命名空间：

- ID 必须唯一；
- 依赖必须存在；
- 不得自依赖；
- 不得形成循环；
- 剧本任务可以依赖 Agent 任务；
- 剧本任务不进入单个 Agent 的待执行任务队列。

## 5. 输出合同

### 5.1 列表

```json
{
  "scene_key": "demo",
  "title": "Demo"
}
```

### 5.2 详情与批量解析

详情和批量解析中的 `definition` 必须包含：

```text
scene_key
title
description
environment
agents
skills
tools
tasks
topology
validation
```

`validation.schema_version` 固定为 `agentnetwork-scene.v2`。`ValidationResult` 的默认版本也必须是 v2，避免未显式传值时回退到旧版本标签。

## 6. 失败语义

- 目录缺少 v2 必需入口：不发现；按名称读取时返回剧本不存在或缺少 v2 文件；
- `env.py` 不是声明式字面量：返回 `ENV_SOURCE_INVALID`；
- 任一严格校验错误：抛出 `SceneValidationError`，一次返回全部可定位问题；
- 上传后校验失败：删除正式目录、文件注册记录和临时解压数据；
- 批量操作：逐项隔离失败，不影响同批其他剧本。

## 7. 旧格式边界

权威运行时读取链路不得读取：

```text
meta_and_roles.json
instances_and_skills.json
network_topology.json
```

旧格式解析只允许存在于 `scripts/migrate_scenes_v2.py` 及其迁移测试，用于把历史输入一次性转换为 v2。`SceneStorage`、`SceneManager`、正式 API、Dashboard 和 MCP 不得恢复 v1 fallback。大型执行模块中遗留但不再作为公开入口调用的历史辅助实现，由 `agent_network.api` 包入口覆盖；后续拆分执行模块时直接删除，不得重新接入路由或调用链。

## 8. 实现映射

- `agent_network/scene_management/scene_storage.py`：发现、安全读取、严格校验和领域模型构造；
- `agent_network/scene_management/validator_v2.py`：v2 严格校验；
- `agent_network/scene_management/models.py`：v2 校验结果与包含 `environment` 的详情模型；
- `agent_network/scene_management/scene_manager.py`：批量解析结果投影；
- `agent_network/api/managed_simulations.py`：正式 HTTP 剧本接口；
- `agent_network/api/__init__.py`：执行模块公开辅助读取的 v2 边界覆盖；
- `tests/test_scene_env_contract.py`、`tests/test_scene_v2_reading.py`：v2 读取、安全解析和无 v1 fallback 回归测试。
