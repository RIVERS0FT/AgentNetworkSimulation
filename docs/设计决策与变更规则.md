# 设计决策与变更规则

本文档记录容易被无意改回的系统决策。除非新的需求明确推翻某项决策，否则后续修改必须保持这些约束。

## ADR-001：Agent 使用直连 HTTP 通信

**决定**：Agent 间消息由 `DirectBus` 直接 POST 目标容器 `/message`。

**原因**：通信路径与真实网络抓包一致，减少中心代理对时延、流量和故障的干扰。

**禁止回退**：

- 不把 `services/message_bus.py` 恢复为默认 relay；
- 不让 MCP `send_message` 调用中心 `/relay`；
- 不通过名称模糊匹配目标。

## ADR-002：拓扑同时定义通信权限和网络 profile

**决定**：同一拓扑边生成对称通信矩阵，并为两端生成出站 `tc` profile。

**影响**：

- 无边即不可通信；
- 有边但网络参数全零时，只建立通信权限；
- 变更拓扑字段必须同步修改严格校验和场景文档。

## ADR-003：网络事实只来自真实 PCAP

**决定**：Agent 容器内 `tcpdump` 是网络证据来源，`network.jsonl` 是真实 packet 的结构化表示。

**禁止回退**：

- 不从 HTTP middleware、Tool 事件或 Agent 消息构造 network record；
- 不把应用层 latency/status 伪装成 TCP/IP 字段；
- 不在没有 PCAP 证据时声称存在数据包。

## ADR-004：日志分为 application、network、system

**决定**：

- `application.jsonl` = Agent 行为 + 传统应用层日志；
- `network.jsonl` = 原始网络层 + 传输层日志；
- `system.jsonl` = 平台调试日志。

**变更规则**：schema 变更采用新版本并一次性迁移 producer、consumer、测试和文档，不保留无需求的旧字段兼容。

## ADR-005：Skill 源文件与 Tool 执行权限分离

**决定**：`skill_refs` 控制源文件读取；`allowed_tools` 控制原子 Tool 执行。

**禁止回退**：

- `srv` 不读 Skill 正文；
- Adapter 不注入完整 `skill_context`；
- Skill 名不注册成 Tool；
- Skill 提及某 Tool 不自动授权。

## ADR-006：逻辑 Agent 与容器槽位分离

**决定**：容器可复用，逻辑身份由每次 `/run` 上下文决定。

**影响**：

- 日志顶层 `agent_id` 必须使用逻辑身份；
- OpenCLAW session 由逻辑 Agent + trace 隔离；
- 分配前必须 reset；
- 容器环境中的静态 ID 仅用于槽位自描述。

## ADR-007：真实后端严格失败，不静默 fallback

**决定**：配置为 OpenCLAW 或 Claude Code 时，SDK/Gateway 缺失应失败。`MOCK_LLM=1` 和显式 `direct_llm` 是独立模式。

**禁止回退**：不得把 direct LLM 当作后端缺失时的透明替代。

## ADR-008：抓包完整性属于实验成功条件

**决定**：抓包启动失败、中途退出、无有效 PCAP 或 hash/manifest 不完整都会影响实验质量。

**原因**：项目目标包括可验证的 Agent 网络流量，不能把抓包当作可选装饰。

## ADR-009：跨层关联是推断，不是因果证明

**决定**：应用事件与 packet 通过 trace、Agent 和时间窗口相关联，输出必须声明其推断性质。

## ADR-010：场景编写文档独立维护

**决定**：`scenes/` 下文档用于指导场景编写；系统架构调整默认只修改 `docs/`。

**例外**：只有当场景输入合同本身发生变化，且任务明确包含场景规范更新时，才同步修改 `scenes/` 文档。

## ADR-011：运行中的剧本资源禁止物理删除

**决定**：在当前运行时仍会按需读取剧本 Skill、场景 Tool 或其他剧本资源的架构下，删除剧本前必须查询运行占用状态。仍被运行中的仿真依赖的剧本不得物理删除。

**原因**：剧本基本结构加载到内存，不代表全部运行依赖已经脱离持久化剧本资源。删除正在使用的剧本可能导致后续 Skill 读取或 Tool 加载失败。

**禁止回退**：

- 不得仅以“剧本结构已加载到内存”为由移除占用检查；
- 不得承诺删除正在运行的原始剧本后仿真必然继续正常运行；
- 不得在没有完整运行快照的情况下允许物理删除被占用剧本。

**可替代设计**：未来若仿真启动时创建包含 Agent 定义、Skill、Tool 和其他运行资源的完整快照，并保证运行过程只访问该快照，可以通过新的设计决策替代本条约束。

## ADR-012：仿真并发上限可扩展

**决定**：当前实现的并发仿真上限为 `1`，但“单仿真”属于当前运行策略，不是永久领域约束。外部接口和目标数据模型必须保留通过 `simulation_id` 和运行引用集合扩展到多个并行仿真的能力。

**当前实现**：

- 控制面使用单一 `simulation_active`、`current_scene_name` 和遗留全局调度状态；
- 启动新仿真前要求当前没有运行中的仿真；
- 剧本占用结果当前最多包含一个运行引用。

**扩展要求**：

- 后续提高并发上限时，将全局状态迁移为按 `simulation_id` 索引的运行注册表；
- Agent 注册表、事件队列、容器分配、通信矩阵、运行配置、日志、PCAP、manifest 和 network profile 必须按仿真实例隔离；
- 状态查询、停止、日志与结果接口必须能够指定 `simulation_id`；
- 剧本占用查询继续返回运行引用集合，不把单一当前剧本字段作为长期接口；
- 并发上限应由配置或容量策略控制，默认值可以保持为 `1`。

**禁止固化**：

- 不得把当前单例全局状态直接定义为永久外部接口；
- 不得设计只能返回一个占用仿真的剧本删除保护合同；
- 不得在扩展多仿真时共享可变 Agent 状态、容器槽位、事件队列或实验产物目录。

## ADR-013：SceneDefinition 分别保存 Skill 与 Tool 定义

**决定**：目标 `SceneDefinition` 必须分别保存 `SkillDefinition[]` 与 `ToolDefinition[]`。Agent 仅通过 `skill_refs` 和 `allowed_tools` 引用对应定义，不使用统一 `CapabilityReference` 混合表示两者。

**原因**：

- Skill 是供后端按需读取的源文件或包；
- Tool 是通过运行时注册并受执行授权控制的原子操作；
- 两者的生命周期、校验规则、权限语义和运行路径不同。

**模型要求**：

- `SceneDefinition.skills` 保存剧本可用 Skill 定义；
- `SceneDefinition.tools` 保存剧本可用 Tool 定义；
- `AgentDefinition.skill_refs` 必须引用 `skills` 中存在的 Skill；
- `AgentDefinition.allowed_tools` 必须引用 `tools` 中存在的 Tool；
- `SceneSummary` 分别返回 Skill 列表与 Tool 列表；
- 剧本名称列表不返回 Skill 或 Tool 数据。

**禁止回退**：

- 不得将 Skill 和 Tool 合并成统一能力定义集合；
- 不得使用同一个枚举字段区分 Skill 与 Tool 后再共享相同结构；
- 不得因为 Skill 文档提及 Tool 就自动创建 Tool 授权；
- 不得只校验 Agent 引用字符串而不校验对应定义集合是否存在。

**当前实现差距**：当前代码只在 `AgentDef` 中保存 Skill 引用和 Tool 授权，尚未在 `SceneDefinition` 中构造两类定义集合。实现该目标设计时应同步修改解析器、校验器、详情模型和测试。

## ADR-014：仿真运行配置归属于仿真实例

**决定**：`SimulationRuntimeConfig` 归属于 `SimulationRun`，不属于 `SceneDefinition`、`SceneResource`、`SceneSummary` 或 `SceneListItem`。

**运行配置包括**：

- `duration_seconds`；
- `agent_timeout_seconds`；
- `idle_timeout_seconds`；
- `network_mode`；
- `seed`；
- `resource_limits`；
- 后续新增的其他事件调度、终止或运行参数。

**模型要求**：

- `SceneDefinition` 只保存静态剧本内容；
- 创建仿真时显式构造 `SimulationRuntimeConfig`；
- 运行配置按 `simulation_id` 保存；
- 事件调度和终止逻辑只读取当前仿真实例的运行配置；
- 剧本列表与详情不得返回或修改仿真运行配置；
- 读取剧本不得设置当前剧本、事件队列、调度进度或全局终止配置。

**禁止回退**：

- 不得将运行时持续时间、执行超时、资源限制、网络模式或随机种子加回 `SceneDefinition`；
- 不得在 `SceneSummary` 或 `SceneListItem` 中增加 `runtime_config`；
- 不得把剧本元数据中的运行参数作为仿真实例的隐式权威配置；
- 不得在剧本解析函数中修改全局仿真状态；
- 不得因为当前只允许一个仿真运行，就把运行配置设计为无 `simulation_id` 所有权的全局合同。

**当前实现差距**：当前剧本加载流程仍会读取旧调度计数字段并写入全局终止配置。后续实现应将这些行为迁移到仿真创建流程，并删除从剧本隐式覆盖运行配置的兼容逻辑。

## ADR-015：剧本管理与仿真运行数据模型分域

**决定**：剧本管理数据模型只包含静态剧本内容和上传、下载、删除、详情、列表过程数据。`SimulationRun`、`SimulationRuntimeConfig`、`SimulationEvent`、`SimulationUsageReference`、`SceneUsageStatus`、`SimulationState` 及其他运行时数据统一归属于仿真编排领域。

**接口边界**：

- 剧本删除通过 `IF-SIM-01` 查询目标剧本是否存在活动占用；
- 面向剧本管理模块的最小结果为 `has_active_usage: boolean`；
- 仿真实例、事件队列、运行引用集合、运行状态详情由仿真管理模块内部维护或通过独立仿真接口提供；
- 剧本管理批量删除只记录 `SCENE_IN_USE` 等处理结果，不复制仿真运行详情。

**禁止回退**：

- 不得在《剧本管理数据模型》中定义仿真实例、运行配置、事件队列、运行引用或运行状态；
- 不得让 `SceneResource`、`SceneDefinition`、`SceneSummary`、`SceneListItem` 或 `BatchSceneOperation` 持有仿真运行对象；
- 不得为了删除保护把仿真注册表结构泄漏给剧本管理模块；
- 不得在剧本管理类图中混入运行时类；
- 不得将仿真领域模型复制一份到剧本领域形成双重权威。

**多仿真扩展**：仿真管理模块内部的占用状态仍使用运行引用集合，以支持未来多个仿真并行运行；剧本管理接口继续只依赖稳定的占用判断语义。

## ADR-016：剧本名称列表与详情模型分离

**决定**：剧本列表使用 `SceneListItem`，单个剧本详情使用 `SceneSummary`。删除 `ScenePreview`，不再保留独立预览模型。

**列表合同**：

- `SceneListItem` 只包含 `scene_key` 和 `title`；
- 前端列表只显示 `title`；
- `scene_key` 仅用于查看详情、下载和删除时定位剧本；
- 列表不返回描述、状态、数量统计、Agent、Skill、Tool、拓扑、校验结果、文件大小或更新时间；
- 列表固定按照 `SceneResource.created_at` 从早到晚输出；
- 不提供用户可配置的排序字段。

**详情合同**：

- `SceneSummary` 是单个剧本的完整静态详情；
- `SceneSummary` 包含基本信息、资源状态、校验状态、Agent、Skill、Tool、拓扑和可选校验结果；
- 查看详情必须是只读操作，不得产生仿真运行副作用。

**禁止回退**：

- 不得重新创建 `ScenePreview`；
- 不得同时维护 `ScenePreview` 和 `SceneSummary` 两套详情结构；
- 不得让列表接口返回完整 `SceneSummary`；
- 不得为了显示名称而逐个解析完整剧本详情；
- 不得在列表中显示除剧本名称之外的业务信息。

**当前实现差距**：当前列表和单个剧本读取接口尚未完全遵循该合同。实现时应统一列表返回 `SceneListItem[]`、详情返回 `SceneSummary`，并删除所有 `ScenePreview` 生产、消费和测试代码。

## ADR-017：剧本固定以目录形式存储

**决定**：平台内部的剧本资源固定以目录形式存储。目录是存储合同，不是可切换格式。

**模型要求**：

- 一个 `scene_key` 对应一个剧本目录；
- `SceneResource.storage_locator` 定位目录；
- 不定义 `SceneResource.format`；
- 不定义 `SceneResourceFormat` 枚举；
- 下载归档只作为传输输出，不改变内部目录存储合同。

**禁止回退**：

- 不得恢复仅含 `folder` 一个值的格式枚举；
- 不得在 `SceneSummary`、列表项或 API 中返回格式字段；
- 未来支持新组织形式时必须重新设计上传、校验、定位、持久化和下载合同，不能只增加枚举值。

## ADR-018：剧本列表为无条件全量列表

**决定**：前端剧本列表永远展示全部已保存剧本。列表接口无请求体、无查询参数，直接返回全部 `SceneListItem[]`。

**列表合同**：

- 不定义 `SceneQuery`；
- 不提供关键字搜索；
- 不提供资源状态或校验状态过滤；
- 不提供 `offset`、`limit` 或其他分页参数；
- 不提供可配置排序；
- 返回全部已保存剧本，固定按 `SceneResource.created_at` 从早到晚排列；
- 前端只显示 `SceneListItem.title`。

**禁止回退**：

- 不得重新创建 `SceneQuery`；
- 不得在列表接口增加搜索、过滤、分页或排序参数；
- 不得把列表拆成分页结果或部分结果；
- 不得让前端为了获取全部剧本而循环请求多页；
- 不得为了列表展示逐个解析完整 `SceneSummary`。

**当前实现差距**：当前列表接口可能仍接受或处理查询参数。实现时应删除相关请求模型、路由参数、过滤分页逻辑和测试，改为一次返回全部 `SceneListItem[]`。

## ADR-019：仿真采用持续时间约束的事件驱动调度

**决定**：目标仿真调度固定采用 `event_driven` 模式。事件到达且目标 Agent 可用时立即调度，不以固定执行次数、停滞计数或调度循环编号划分仿真进程。

**运行合同**：

- 初始任务、Agent 消息、后续任务和系统触发统一转换为 `SimulationEvent`；
- 事件队列按 `simulation_id` 隔离；
- 同一逻辑 Agent 默认单任务串行，不同 Agent 可在资源限制范围内并行；
- 无事件时阻塞等待，不进行固定周期空转；
- `duration_seconds` 是强制持续时间上限；
- `idle_timeout_seconds` 用于事件队列为空且无执行中任务时的空闲终止；
- Agent 执行超时、资源限制、抓包健康和用户停止都属于独立终止或失败条件。

**禁止回退**：

- 不得重新引入 `max_rounds`、`stalemate_rounds`、`current_round`、`current_turn`、`current_max_rounds`、`round`、`tick`、`total_rounds` 或 `event_driven_rounds` 作为目标合同；
- 不得以固定次数循环表达仿真持续时间；
- 不得从剧本元数据隐式读取旧调度字段；
- 不得让 AgentContext、API、Dashboard、日志或 manifest 继续把旧调度字段作为权威状态。

**完整说明**：参见 [ADR-019-持续时间约束的事件驱动仿真.md](ADR-019-持续时间约束的事件驱动仿真.md)。

**当前实现差距**：当前代码仍有固定次数循环和旧上下文字段。迁移时必须同步修改 API、运行模型、事件调度器、Dashboard、日志、manifest、质量审计和测试，并直接删除旧合同，不保留双轨兼容。

## 通用变更规则

1. 每次新工作先阅读 `docs/README.md`、相关设计文档和代码。
2. 代码与设计文档必须在同一 PR 中保持一致；纯目标设计变更必须明确标注当前实现差距。
3. 当前处于初期开发阶段：优先直接迁移，不为旧实现增加兼容分支。
4. 删除架构后，删除所有文档、compose、测试和 Dashboard 中的旧引用。
5. 新增字段、接口或后台任务时，写明所有权、生命周期、失败语义和观测方式。
6. 完成后检查 diff，确保没有修改任务范围外的 `scenes/` 文档。
7. 无异常且检查通过时，直接合并。
