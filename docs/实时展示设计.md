# 实时展示设计

> 状态：目标设计。本文定义 `SR-WEB-01～04` 的实时数据边界、前端展示合同、断线恢复和剧本自定义面板安全约束。

## 1. 需求范围

| SR-ID | 需求名称 | 设计目标 |
|---|---|---|
| SR-WEB-01 | 实时展示：日志 | 实时展示当前仿真中 Agent 已发生的历史行为动作。 |
| SR-WEB-02 | 实时展示：Agent 拓扑 | 实时展示 Agent 信息悬浮框、通信拓扑、通信流、仿真进度和状态动画。 |
| SR-WEB-03 | 实时展示：Token 消耗量 | 实时展示当前仿真 Token 累计消耗随时间的变化。 |
| SR-WEB-04 | 实时展示：剧本可视化 | 按剧本编写人员声明的面板定义展示受控的实时可视化内容。 |

## 2. 总体数据流

```text
SimulationManager / EventScheduler / Agent Runtime / LogManager
                         |
                         | 领域事件、状态快照、日志增量
                         v
                 Realtime View Adapter
                         |
                         | WebSocket 或 SSE
                         v
                    Web Dashboard
        +----------------+----------------+
        |                |                |
     日志时间线       Agent 拓扑       Token 累计图
        |                                 |
        +----------- 剧本自定义面板 ------+
```

实时展示层只投影已经存在的领域事实，不参与仿真调度，不修改 Agent 状态，也不生成网络事实。

## 3. 实时消息合同

每条实时消息至少包含：

```json
{
  "type": "snapshot|delta|heartbeat|error",
  "simulation_id": "sim_xxx",
  "sequence": 123,
  "timestamp": "2026-07-20T12:00:00.000Z",
  "payload": {}
}
```

约束：

1. `simulation_id` 必须明确，禁止用无实例归属的全局消息作为长期合同。
2. `sequence` 在同一仿真内单调递增，用于去重和断线补偿。
3. 首次连接或重连先返回 `snapshot`，再发送 `delta`。
4. 前端发现序号缺口时必须请求新快照，不能继续假设状态完整。
5. `heartbeat` 只表示连接存活，不代表仿真仍在运行。
6. `error` 必须包含稳定错误码和用户可理解的说明。

## 4. SR-WEB-01：实时日志

### 4.1 数据来源

- Agent 行为动作：`application.jsonl` 及 `LogManager` 的实时索引；
- 平台故障和调试信息：`system.jsonl`；
- 网络数据包：`network.jsonl`，只在网络视图中展示，不混入 Agent 行为时间线。

### 4.2 展示模型

日志项至少包含：

| 字段 | 描述 |
|---|---|
| `timestamp` | 事件时间 |
| `agent_id` | 逻辑 Agent 标识 |
| `event` | 行为事件类型 |
| `action` | 可读动作摘要 |
| `target` | 目标 Agent、Tool、Skill 或外部服务 |
| `status` | 进行中、成功、失败等状态 |
| `trace_id` | 跨层追踪标识 |
| `details` | 可展开的结构化详情 |

前端按 `session_id + sequence` 或稳定事件 ID 去重。历史动作只追加，不因状态刷新而覆盖已经发生的事件。

## 5. SR-WEB-02：Agent 拓扑

### 5.1 权威数据

- 节点：当前仿真实例的逻辑 Agent 定义、运行状态和容器分配摘要；
- 边：当前仿真实例的 `comm_matrix` 与拓扑网络参数；
- 通信流：A2A 消息事件或真实抓包中能够验证的通信事实；
- 仿真状态：`SimulationRun.state`；
- 进度：墙钟时间、`duration_seconds` 和提前完成状态。

### 5.2 Agent 信息悬浮框

悬浮框至少显示：

- Agent 名称、标识、角色；
- 当前运行状态；
- 当前任务摘要；
- 已授权 Skill 和 Tool 数量；
- 最近活动时间；
- 当前容器槽位摘要；
- 错误或超时状态。

悬浮框不得展示密钥、完整 prompt、未脱敏凭证或超出当前用户权限的日志正文。

### 5.3 通信流动画

1. 动画由具体事件触发，并绑定 `trace_id` 或消息 ID。
2. 动画方向必须与消息源和目标一致。
3. 多目标发送按实际逐个发送顺序展示，不显示广播效果。
4. 没有真实事件时不得生成装饰性通信流。
5. 动画结束不代表消息业务成功，成功状态由 A2A 或任务回调事件决定。

### 5.4 进度与状态动画

```text
progress = min(elapsed_seconds / duration_seconds, 1)
```

- `configured`：已配置、未启动；
- `starting`：准备资源和抓包；
- `running`：正常运行；
- `stopping` / `force_stopping`：停止中；
- `completed` / `stopped` / `force_stopped` / `failed`：终态。

全局任务提前完成时，进度条直接进入完成态，并展示实际结束原因，而不是继续等待最大持续时间。

## 6. SR-WEB-03：Token 累计图

### 6.1 数据来源

Token 使用量来自 LLM 调用或运行时完成事件中的可验证 metrics。至少识别：

- prompt/input token；
- completion/output token；
- cache hit token；
- cache miss token；
- 总 token。

### 6.2 累计规则

1. 每个调用使用稳定事件 ID 去重。
2. 各累计序列不得下降。
3. 数据点属于一个明确的 `simulation_id`。
4. 切换仿真实例时清空当前视图并加载目标实例快照。
5. 缺失 token 数据时显示未知或部分统计，不使用字符数随机换算成权威 token。
6. 图表时间轴使用事件时间，必要时同时保留接收时间用于诊断延迟。

建议快照：

```json
{
  "simulation_id": "sim_xxx",
  "points": [
    {
      "timestamp": "2026-07-20T12:00:01.000Z",
      "prompt_total": 1200,
      "completion_total": 300,
      "cache_hit_total": 500,
      "cache_miss_total": 700,
      "grand_total": 1500
    }
  ]
}
```

## 7. SR-WEB-04：剧本自定义面板

### 7.1 声明式面板

剧本只能声明面板配置，不得上传并执行任意 JavaScript、HTML 或浏览器扩展代码。

建议结构：

```yaml
visualization:
  version: "1"
  title: "任务执行面板"
  layout:
    columns: 12
  widgets:
    - id: task-status
      type: status_table
      title: "任务状态"
      data_source: simulation.tasks
      fields: [task_id, agent_id, state, updated_at]
```

### 7.2 白名单

平台维护：

- 组件类型白名单；
- 字段和聚合函数白名单；
- 数据源白名单；
- 交互动作白名单；
- 最大组件数量、数据点数量和刷新频率限制。

第一版建议组件：

- 指标卡；
- 状态表；
- 时间序列图；
- 柱状图；
- 事件列表；
- Agent 状态矩阵；
- 只读 Markdown 说明。

### 7.3 数据源

允许的数据源只能来自平台已校验的投影，例如：

- `simulation.summary`；
- `simulation.tasks`；
- `simulation.agents`；
- `simulation.token_usage`；
- `application.events`；
- `network.aggregates`。

面板不能直接读取宿主文件系统、浏览器存储、任意 URL、密钥或其他仿真实例数据。

## 8. API 与 WebSocket 映射

建议保持一个统一实时入口：

```text
GET /api/simulations/{simulation_id}/realtime/snapshot
WS  /ws/simulations/{simulation_id}
```

消息主题可以包括：

- `simulation.status`；
- `simulation.progress`；
- `agent.status`；
- `application.event`；
- `communication.event`；
- `token.usage`；
- `visualization.data`。

现有全局 `/ws` 可以作为迁移入口，但目标合同必须按 `simulation_id` 隔离。

## 9. 当前实现映射

- `services/server.py`：提供当前 `/ws` 连接、Agent 状态、日志和拓扑快照；
- `web/public/dashboard.js`：日志归一化、去重、实时渲染和 Token 累计图；
- `agent_network/simulation_management/state.py`：当前控制面共享状态；
- `agent_network/log_management/log_manager.py`：日志实时索引；
- `agent_network/simulation_management/simulation_manager.py`：仿真实例状态与持续时间。

当前实现仍使用部分全局状态。后续迁移必须按 `simulation_id` 隔离实时快照和增量，不得把单仿真假设固化为永久前端协议。

## 10. 验收要点

1. 仿真运行时产生 Agent 行为后，日志视图在可接受延迟内出现且不重复。
2. Agent 状态改变时，节点状态和悬浮框同步更新。
3. A2A 消息发生时，只在真实源节点和目标节点之间展示一次通信流。
4. 达到最大持续时间或全局任务完成时，进度和终态原因一致。
5. Token 累计图在断线重连、重复消息和切换仿真后保持正确。
6. 剧本面板 schema 非法、使用未知组件或越权数据源时，平台拒绝加载并返回明确错误。
7. 任意剧本资源均不能执行上传的浏览器脚本。
