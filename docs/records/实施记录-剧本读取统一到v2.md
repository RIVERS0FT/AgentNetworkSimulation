# 实施记录：剧本读取统一到 v2

- 日期：2026-07-23
- 关联：ADR-036、ADR-037

## 问题

存量剧本已经迁移到 `Agents.json`、`topology.json` 和 `env.py`，但运行时仍残留旧格式读取路径：

- `SceneStorage` 仍识别并构造 v1 剧本；
- 执行模块辅助 `list_scenes()` 只查找 `meta_and_roles.json`；
- 执行模块辅助 `read_scene()` 只返回三份旧 JSON；
- 批量解析的定义结果没有返回 `environment`；
- `ValidationResult` 默认版本仍是 `agentnetwork-scene.v1`；
- 多个回归测试仍以旧格式作为正常输入。

迁移后这些路径会导致剧本列表为空、读取结果缺失、环境数据丢失或错误标记为 v1。

## 决定

1. 运行时只接受完整 v2 目录。
2. `SceneStorage` 删除 v1 文件发现、校验器和领域模型构造分支。
3. 文件注册表中的旧目录记录不进入列表，也不能被按名称读取。
4. 正式 API 和执行模块公开辅助读取统一委托 `get_scene_storage()`。
5. 详情和批量解析都固定返回 `environment`。
6. `ValidationResult` 默认 schema 版本改为 v2。
7. 删除不再使用的旧 `SceneValidator`。
8. 删除只会从旧三文件生成 `agents.json` 的过期 `extract_scene_agents.py`。
9. 旧格式读取只保留在 `scripts/migrate_scenes_v2.py` 和迁移测试中。

## 实现映射

- `agent_network/scene_management/scene_storage.py`：v2 发现、读取、校验和领域模型构造；
- `agent_network/scene_management/scene_manager.py`：批量解析包含 `environment`；
- `agent_network/scene_management/models.py`：v2 默认 schema 和详情环境字段；
- `agent_network/scene_management/__init__.py`：移除旧校验器导出；
- `agent_network/api/__init__.py`：覆盖执行模块公开列表和详情辅助函数，使其统一读取 `SceneStorage`；
- 删除 `agent_network/scene_management/validator.py`；
- 删除 `scripts/extract_scene_agents.py`；
- 新增 `tests/test_scene_v2_reading.py`；
- 将场景管理、构建边界、文件管理和 Skill 存储测试迁移到 v2 fixture。

## 失败语义

- 缺少任意 v2 必需入口的目录视为不存在的剧本；
- 已注册但已变成旧格式或不完整格式的目录不会出现在列表中；
- v2 内容错误继续通过 `SceneValidationError` 返回完整问题集合；
- 上传包不是完整 v2 时继续回滚正式目录与临时资源。

## 兼容与回滚

不保留 v1 运行时兼容。需要处理历史目录时，先运行迁移脚本。回滚必须整体恢复旧校验器、旧读取分支、旧测试和旧数据合同，不允许只恢复其中一部分形成双重权威。
