# AgentNetwork 系统需求规格

> 状态：权威产品需求基线。本文定义平台当前确认的系统需求（SR）及其设计追踪规则。详细设计、接口、数据模型、实现和测试不得弱化或改变本文需求；如需变更，必须在同一提交中先更新本文及相关设计文档。

## 1. 适用范围

本文覆盖四类平台能力：

- 剧本管理；
- 日志管理；
- 仿真模拟；
- 实时展示。

本文中的“剧本”是已经完成编辑、能够通过平台校验并用于仿真运行的静态场景资源；“日志”至少包括 `application.jsonl` 与 `network.jsonl`，`system.jsonl` 作为平台调试日志继续保留；“实时”表示仿真运行期间通过持续推送或短周期增量同步更新界面，而不是仿真结束后一次性展示。

## 3. 需求解释与边界

### 3.1 剧本管理

1. 批量操作必须逐项返回成功、失败和失败原因，单项失败不得导致其他项回滚。
2. 上传对象必须经过严格校验后才能成为可运行剧本；失败项必须清理临时资源。
3. 删除剧本前必须检查活动仿真占用。被正在运行的仿真使用时不得物理删除。
4. “预览剧本”是仿真开始前的只读查看能力，可以复用 `SceneSummary` 作为返回模型，不要求额外创建 `ScenePreview` 类型；预览不得创建仿真实例、分配容器或修改运行状态。
5. “查询剧本”不是仅返回名称的无条件列表。查询结果至少应展示剧本标识、剧本名称、可用状态和基本信息；详细设计至少支持按名称关键字和状态筛选，是否分页由接口设计决定。
6. 剧本状态应区分资源可见性、校验结果和是否可运行，前端可以展示聚合状态，但不得把正在运行的仿真实例状态写入静态剧本定义。

### 3.2 日志管理

1. 每次仿真必须形成独立日志会话，并至少保存：
   - `application.jsonl`：Agent 行为与传统应用层日志；
   - `network.jsonl`：由真实 PCAP 投影得到的原始网络层与传输层日志。
2. `system.jsonl` 继续作为平台调试日志保留，但不能替代上述两类必需日志。
3. 批量下载、隐藏、显示和删除均须支持跨会话、跨日志类型逐项处理。
4. 隐藏是可见性状态，不等于删除；隐藏日志仍可继续写入，默认查询和下载不得返回隐藏资源。
5. 活动日志会话不得被删除；删除必须保持实验清单和审计结果的一致性。

### 3.3 仿真模拟

1. 仿真配置属于具体仿真实例，不属于静态剧本。
2. 配置至少包括最大持续时间和系统资源限制；资源限制至少覆盖 CPU、内存和最大并行 Agent 数，是否包含进程数限制由运行时能力决定。
3. 运行前必须完成剧本校验、容量校验、资源分配计划、日志会话和抓包准备。
4. 手动停止应先执行优雅停止；超时或明确强停时允许进入强制停止流程。
5. 自动完成条件包括：
   - 达到最大仿真持续时间；
   - 全局任务完成，且没有待处理事件、执行中任务或必须等待的回调。
6. 自动停止后必须执行抓包停止、网络规则清理、日志落盘、manifest 完成和资源释放。

### 3.4 实时展示

1. 实时日志以 `application.jsonl` 事件和运行时增量索引为主要来源，展示 Agent 历史行为动作，不把 `network.jsonl` 记录伪装成 Agent 行为。
2. Agent 拓扑展示必须以当前仿真实例的 Agent 目录和通信矩阵为权威来源；通信流动画只能由真实消息事件或可验证的网络事件触发。
3. 仿真进度按已用墙钟时间相对于 `duration_seconds` 计算，并同时展示当前运行状态；全局任务提前完成时进度可以直接进入完成态。
4. Token 图展示当前仿真的累计消耗，时间序列必须单调不减；统计至少覆盖 prompt、completion 和可识别的 cache token，无法获取时应明确标记未知而不是填充伪造值。
5. 剧本可视化面板由剧本编写人员声明，但平台必须实施 schema 校验、组件白名单、数据源白名单和脚本/HTML 隔离，禁止直接执行任意前端代码。
6. WebSocket、SSE 或轮询只是传输实现，不改变实时展示的业务合同；断线重连后必须通过快照加增量恢复当前视图。

## 6. 变更与防回退规则

1. SR-ID 一经发布不得复用或改变含义；废弃需求保留原 ID，并明确标记替代项。
2. 修改任何 SR 时，必须在同一提交中更新本文、关联详细设计、API/数据模型说明和对应测试计划。
3. 新增实现不得以“当前代码尚未支持”为理由删除或弱化 SR；应标记为“部分实现”或“设计阶段”。
4. 详细设计与本文冲突时，以本文为准，并在同一变更中修正文档冲突。
5. `SR-SCENE-05` 明确取代旧的“只返回全部剧本名称、禁止查询和状态展示”合同。
6. `SR-SCENE-04` 的产品名称统一为“预览剧本”；技术实现可以继续复用只读 `SceneSummary`，不得因此删除预览能力。
7. `SR-LOG-01` 要求的应用层和网络层日志分别对应 `application.jsonl` 与 `network.jsonl`，不得通过合并日志文件或伪造网络记录规避。
8. `SR-WEB-04` 只允许声明式、安全受控的剧本面板，不得演变为执行任意上传脚本。
